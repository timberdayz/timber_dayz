# 备份目录说明

## 目录用途

此目录用于存放项目开发过程中的重要备份文件，包括：

- 架构重构前的代码备份
- 重要功能模块的历史版本
- 配置文件的安全备份
- 数据库迁移前的备份

## 文件组织规范

### 命名规范
- 目录：`YYYYMMDD_功能描述_版本号`
- 文件：`YYYYMMDD_HHMMSS_功能描述_版本号.扩展名`

### 示例
```
backups/
├── 20250828_document_consolidation/
├── 20250814_account_config_cleanup/
├── 20250810_architecture_optimization_cleanup/
└── accounts_config_20250804_180223.json
```

## 清理策略

- **开发备份**：保留最近3个月
- **架构备份**：永久保留
- **配置备份**：保留最近6个月
- **数据备份**：根据重要性决定保留期

## 注意事项

⚠️ **安全提醒**：
- 备份文件可能包含敏感信息
- 不要将包含真实账号/密钥的备份提交到版本控制
- 定期检查并清理过期备份

✅ **最佳实践**：
- 重要操作前先创建备份
- 备份文件添加详细说明
- 使用有意义的文件名
- 定期验证备份完整性
