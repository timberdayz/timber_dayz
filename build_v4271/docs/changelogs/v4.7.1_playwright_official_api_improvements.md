#### 2.3 `_extract_target_url_from_actions()` - 新增辅助方法 ⭐

**功能**: 从录制的操作中智能提取目标 URL 特征

```python
def _extract_target_url_from_actions(self) -> Optional[str]:
    """
    从录制的操作中提取目标 URL 特征
    
    示例:
        输入: https://erp.example.com/orders/list?date=2024
        输出: /orders
    """
    if not self.recorded_actions:
        return None
    
    # 查找最后一个 navigate 动作
    for action in reversed(self.recorded_actions):
        if action.get('action') in ['navigate', 'goto']:
            url = action.get('url', '')
            
            try:
                from urllib.parse import urlparse
                parsed = urlparse(url)
                path = parsed.path
                
                # 提取第一个有意义的路径段
                if path and len(path) > 1:
                    parts = [p for p in path.split('/') if p]
                    if parts:
                        return '/' + parts[0]  # /orders/list → /orders
            except Exception:
                pass
    
    return None
```

## 🎓 Playwright 官方 API 最佳实践总结

### 1. 导航和等待

```python
# ⭐ 官方推荐：page.goto() 已内置等待
await page.goto(url, wait_until='domcontentloaded')  # 基础等待
await page.wait_for_load_state('networkidle')        # 网络空闲（SPA推荐）

# ❌ 不推荐：显式 wait_for_selector（旧API）
await page.wait_for_selector(selector)  # 可用 locator 替代
```

### 2. 元素定位

```python
# ⭐ 官方推荐：使用 Locator API（自带自动等待）
await page.locator(selector).click()
await page.get_by_role("button", name="登录").click()  # 最佳方式
await page.get_by_text("欢迎").is_visible()

# ❌ 不推荐：手动等待元素
await page.wait_for_selector(selector)
element = await page.query_selector(selector)
await element.click()
```

### 3. 选择器优先级

1. ⭐⭐⭐ `role=button[name='Login']` - 官方推荐（可访问性友好）
2. ⭐⭐ `text='Welcome'` - 文本定位（稳定）
3. ⭐⭐ `label='Email'` - 表单元素（语义化）
4. ⭐ `placeholder='Enter email'` - 占位符定位
5. CSS/XPath - 降级方案（易碎）
