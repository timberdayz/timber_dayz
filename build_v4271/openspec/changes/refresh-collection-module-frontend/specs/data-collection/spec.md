# 数据采集能力 - 前端整体修改增量

## ADDED Requirements

### Requirement: 数据采集管理界面 SHALL 支持手动与自动采集的完整操作与可观测

系统 SHALL 在数据采集管理前端提供：采集任务列表与进度/状态展示、快速采集与配置执行（手动触发）、采集配置与定时（自动触发）、任务详情与步骤时间线、采集历史与重试；所有展示与操作 SHALL 与后端 API 契约一致。

#### Scenario: 手动发起采集并在任务列表观测

- **WHEN** 用户在「采集任务」页的快速采集区域选择平台、账号、数据域、日期范围（及可选调试模式）并点击「开始采集」
- **THEN** 前端调用 POST /collection/tasks 创建任务，参数与表单一致
- **AND** 新任务出现在采集任务列表中，列表展示任务 ID、平台、账号、数据域、进度、当前步骤或数据域、状态、文件数、创建时间
- **AND** 用户可通过筛选（全部/运行中/排队中/已完成/失败）查看任务；进度与状态与后端返回一致，不出现错误展示（如始终 0%）

#### Scenario: 从采集配置执行并观测

- **WHEN** 用户在「采集配置」页某条配置上点击「执行」
- **THEN** 前端基于该配置调用创建任务接口，新任务在「采集任务」列表中可见
- **AND** 用户可在采集任务列表查看该任务的进度与状态，并打开任务详情或日志

#### Scenario: 自动定时采集任务可见

- **WHEN** 用户在「采集配置」中为某条配置启用定时并设置执行时间（cron 或预设）
- **THEN** 前端将定时设置提交至后端调度接口并保存
- **AND** 定时触发的任务在「采集任务」列表中展示，状态与进度与后端一致

#### Scenario: 任务详情与步骤时间线可查看

- **WHEN** 用户在采集任务列表（或采集历史）点击「任务详情」或等效入口
- **THEN** 前端展示任务元信息（平台、账号、状态、时间、总耗时、error_message、completed_domains、failed_domains）
- **AND** 前端调用任务日志接口（GET /collection/tasks/:taskId/logs），按 timestamp 排序展示步骤时间线；若日志含 details（step_id、component、data_domain、success、duration_ms、error），展示为步骤名与成功/失败，失败步骤高亮且 details.error 可展开
- **AND** 现有「查看日志」弹窗（原始日志流）保留，与步骤时间线互补

#### Scenario: 采集历史与重试

- **WHEN** 用户在「采集历史」页查看列表或点击某条「详情」
- **THEN** 列表与详情数据与后端 GET /collection/history 及任务详情/日志接口一致
- **AND** 失败任务的「重试」调用 POST /collection/tasks/:taskId/retry，成功后列表刷新或新任务在采集任务列表可见

#### Scenario: 错误与空态处理

- **WHEN** 任一采集相关接口返回错误或数据为空
- **THEN** 前端展示后端返回的 message 或合理文案，不展示未定义或错误字段名
- **AND** 列表与详情具备明确加载态与空态，避免误导用户
