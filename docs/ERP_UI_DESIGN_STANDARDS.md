## 🎯 核心设计原则

### 1. 宽度控制原则

**黄金法则**: 页面内容宽度应该适应用户屏幕，但不应过宽导致阅读困难

**推荐标准**:
```
超宽屏（>1920px）: 内容最大1800px，居中显示
大屏（1440-1920px）: 内容最大1600px，居中显示
中屏（1024-1440px）: 内容最大1400px，居中显示
小屏（<1024px）: 内容100%自适应
```

**原因**:
- 视线扫描距离：人眼舒适视角约120度
- 阅读效率：行长度过长导致阅读效率下降
- 操作便利：鼠标移动距离不宜过长

### 2. 表格宽度处理

**问题**: 表格列数多时（>12列），水平宽度失控

**企业级解决方案**:
```
方案1: 固定关键列 + 横向滚动（SAP标准）⭐⭐⭐
  - 左侧固定：ID、SKU、名称（2-3列）
  - 中间滚动：分类、价格、库存等（可滚动）
  - 右侧固定：操作列（1列）

方案2: 列显示控制（Oracle标准）⭐⭐
  - 用户选择显示哪些列
  - 保存用户偏好
  - 预设方案（简洁/标准/完整）

方案3: 响应式列隐藏（移动端）⭐
  - 小屏自动隐藏次要列
  - 点击行展开查看全部
```

## 📊 我们的实施标准（v4.9.3）

### 页面分类和宽度标准

| 页面类型 | 容器类 | 最大宽度 | 说明 |
|---------|--------|---------|------|
| **数据浏览器** | `erp-wide-container` | 1800px | 需要显示更多列 |
| **产品管理** | `erp-page-container` | 1600px | 标准列表页 |
| **字段映射** | `erp-page-container` | 1600px | 配置页面 |
| **看板页面** | `erp-content-container` | 1400px | 图表为主 |
| **表单页面** | `erp-form-container` | 800px | 表单不宜过宽 |

### 表格固定列标准

#### 数据浏览器
```
固定左侧（2列）:
  - 第1列（ID/主键）: 100px
  - 第2列（名称/SKU）: 200px

滚动中间:
  - 其他所有字段: 150px

固定右侧:
  - 无（数据查看为主，无操作列）
```

#### 产品管理
```
固定左侧（3列）:
  - 产品图片: 100px
  - SKU: 180px
  - 产品名称: 250px

滚动中间:
  - 分类: 120px
  - 品牌: 120px
  - 价格: 120px
  - 库存: 100px
  - 销量: 100px
  - 浏览量: 100px

固定右侧（1列）:
  - 操作: 180px
```

#### 字段映射
```
固定左侧（1列）:
  - 原始字段: 250px

滚动中间:
  - 示例数据: 200px
  - 数据库列名: 200px
  - 标准字段: 200px
  - 置信度: 100px
  - 匹配方式: 120px

固定右侧:
  - 无
```

## 🎁 优化效果

### 优化前问题

**宽度失控**:
- 40列的表格：总宽度6000px
- 超宽屏也看不全（需要横向滚动很远）
- 关键列滚动后看不见

**用户体验差**:
- 视线左右扫描距离过长
- 找不到操作按钮（滚动到最右侧）
- 对比数据困难（左右列距离太远）

### 优化后效果

**宽度控制**:
- 页面最大1600-1800px，超宽屏居中显示
- 关键列固定（SKU、产品名始终可见）
- 操作列固定（操作按钮始终可见）

**用户体验优**:
- 视线扫描舒适（不超过1600px）
- 关键信息始终可见（固定列）
- 横向滚动平滑（只滚动次要列）
- 单元格Tooltip（悬停查看完整内容）
