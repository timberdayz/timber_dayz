## ⚠️ 3个最重要的规则

### 规则1: 不要改对方的文件
```
Agent A（Cursor）只改：
  models/, migrations/, services/etl*, services/field_mapping/

Agent B（Augment）只改：
  frontend_streamlit/, utils/, services/currency*, services/data_query*

查看完整规则: docs/FILE_ISOLATION_RULES.md
```

### 规则2: 共享文件先定义接口
```
如果要修改 services/data_query_service.py:
  1. 先在 docs/API_CONTRACT.md 中定义接口
  2. 双方确认后再实现
  3. Git提交时注明"⚠️ 接口变更"
```

### 规则3: 不要改冻结区域
```
❌ 禁止修改：
  - modules/apps/collection_center/（数据采集模块）
  - frontend_streamlit/pages/10_数据采集中心.py
```

## 💡 新手提示

### 如果你是编程新手
1. **不要怕问AI**：把AI当成你的导师，不懂就问
2. **从简单开始**：先让代码跑起来，再考虑优化
3. **频繁测试**：每写一点代码就测试一下
4. **及时提交**：每2-3小时提交一次代码（防止丢失）

### 常用的Agent指令

#### 让Agent写代码
```
"帮我创建一个Excel解析器类，支持.xlsx和.xls格式，
 参考项目中类似的代码风格"
```

#### 让Agent修复错误
```
"运行这段代码报错了：
[粘贴错误信息]
请帮我修复"
```

#### 让Agent解释代码
```
"这段代码是做什么的？
[粘贴代码]
能详细解释一下吗？"
```

#### 让Agent优化性能
```
"这段代码运行很慢：
[粘贴代码]
请帮我优化性能，目标是提速10倍"
```

## 📞 遇到问题？

### 技术问题
- 看不懂代码 → 问AI："这段代码是做什么的？"
- 不知道怎么实现 → 问AI："如何实现XXX功能？"
- 代码报错 → 问AI："[粘贴错误]，帮我修复"

### 流程问题
- 不知道改哪个文件 → 查看`FILE_ISOLATION_RULES.md`
- 不知道今天做什么 → 查看`AGENT_A/B_HANDBOOK.md`
- 不知道如何切换工具 → 查看上面的"7天时间表"

### 进度问题
- 任务完成不了 → 立即砍功能，保证核心能跑
- 遇到难题卡住 → 超过30分钟就跳过或简化
- 不确定优先级 → 核心 > 辅助，功能 > 美观
