# 绩效赛马与预警口径说明

## 1. 目的

本文用于统一说明当前绩效模块中的赛马、预警、挂店人继承和提成奖惩口径，供 HR、管理层、财务和系统维护人员对照使用。

本文是当前运行口径的正式说明。若与历史设计草稿存在差异，以本文和系统实际实现为准。

## 2. 核心原则

- 店铺绩效先按店铺维度计算，作为赛马基础。
- 公司所有满足条件的店铺进入同一个月度赛马池。
- 挂店人提成奖惩不再单独按个人档位发放，而是继承所挂店铺的赛马结果。
- 绩效分用于评价责任贡献；赛马系数用于提成奖惩。
- 观察池、黄牌、红牌、淘汰评估属于经营治理机制，不改变月结中心“汇总消费上游结果”的定位。

## 3. 店铺绩效口径

店铺绩效按月计算，当前由四个维度构成：

- 销售分
- 毛利分
- 重点产品分
- 运营分

店铺总分：

```text
店铺总分 = 销售分 + 毛利分 + 重点产品分 + 运营分
```

## 4. 赛马池规则

### 4.1 正式赛马池

满足以下条件的店铺进入正式赛马池：

- 当月为有效店铺
- 当月四个维度都完成计算
- 当月实际经营天数大于等于 15 天

### 4.2 观察池

不满足正式赛马池条件的店铺进入观察池：

- 观察池店铺仍然展示绩效分
- 观察池店铺不参与正式排名
- 观察池店铺不产生正式赛马系数

## 5. 店铺赛马系数

正式赛马池中的店铺先按公司总榜排名分档，再叠加绝对分数底线封顶。

### 5.1 排名分档

- 前 20%：1.2
- 20% ~ 50%：1.0
- 50% ~ 80%：0.9
- 后 20%：0.8

### 5.2 分数底线封顶

- 店铺绩效分大于等于 80：不封顶
- 店铺绩效分 70 ~ 79.99：最高 1.0
- 店铺绩效分 60 ~ 69.99：最高 0.8
- 店铺绩效分低于 60：直接 0.5

### 5.3 最终公式

```text
最终店铺赛马系数 = min(排名分档系数, 分数底线系数)
```

## 6. 挂店人继承规则

### 6.1 员工绩效分

员工绩效分用于评价责任贡献，当前口径为：

```text
员工最终绩效分 = 挂店店铺绩效承接结果 + 个人加减分
```

其中个人加减分目前包括：

- 考勤自动扣分
- 月度个人绩效调整项
  - 考试
  - 培训检核
  - 人工奖惩

### 6.2 员工提成赛马系数

员工提成奖惩不再单独按个人分数分档，而是按挂店销售权重继承店铺赛马系数：

- 只挂 1 个店：直接继承该店赛马系数
- 挂多个店：按店铺销售权重加权继承多个店的赛马系数

推荐理解方式：

```text
员工赛马系数 = 所挂店铺赛马结果的加权平均
```

## 7. 预警与淘汰评估

### 7.1 预警规则

- 店铺绩效分低于 70：黄牌
- 店铺绩效分低于 60：红牌

### 7.2 淘汰评估规则

当店铺历史已连续两个月红牌，且本月继续红牌时，系统生成“淘汰评估”预警。

### 7.3 说明

- 预警结果写入 `c_class.shop_alerts`
- 预警用于经营治理、整改和淘汰评估
- 预警与赛马系数并行存在，但功能不同

## 8. 提成联动规则

当前提成采用两段式口径：

```text
基础提成 = 利润分配基准 × 可分配利润率 × commission_ratio
实际提成 = 基础提成 × 员工赛马系数
```

其中利润分配基准当前统一为：

```text
profit_basis_amount = orders_profit_amount - a_class_cost_amount
```

## 9. 页面解释口径

当前前端页面中：

- `赛马池`
  - 正式池：参与正式排名与赛马奖惩
  - 观察池：只展示绩效，不参与正式奖惩
- `预警`
  - 黄牌：绩效低于 70
  - 红牌：绩效低于 60
  - 淘汰评估：连续红牌达到升级条件

人员维度下应理解为：

- 绩效分：承接挂店店铺绩效并叠加个人项
- 提成系数：继承挂店店铺赛马结果
