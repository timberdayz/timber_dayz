# ERP 操作培训第一期上线检查清单

## 目的

本清单用于在不等待完整培训系统开发完成的前提下，先跑通 `ERP 操作培训第一期` 的真实试点流程。

试点目标不是一次性把全部培训能力做完，而是先验证以下闭环是否成立：
- 培训内容能否顺利发布
- 员工能否完成学习与考试
- 结果能否回填到 ERP 管理口径
- 主管和培训负责人能否追踪完成情况

## 试点范围

本次试点建议仅覆盖：
- 培训项目：`ERP 操作培训第一期`
- 适用对象：新员工 / 首次使用 ERP 的员工 / 小范围试点用户
- 试点人数：5 到 15 人
- 培训周期：3 到 7 天

## 一、内容准备检查

### 1. 培训包结构是否确认

- [ ] 培训项目名称已确认：`ERP 操作培训第一期`
- [ ] 培训目标已确认
- [ ] 通过标准已确认：完成全部课程 + 考试通过
- [ ] 培训周期已确认
- [ ] 补考规则已确认

### 2. 六个培训模块是否齐备

- [ ] 系统登录与基础导航
- [ ] 任务中心与通知中心
- [ ] 基础数据查询
- [ ] 常用业务页面操作
- [ ] 操作规范与留痕意识
- [ ] 异常与升级处理

### 3. 每个模块材料是否齐备

每个模块至少应有：
- [ ] 1 份图文讲义或 PPT
- [ ] 关键模块截图
- [ ] 明确的讲解重点
- [ ] 对应考试考点

### 4. 培训材料质量是否满足试点要求

- [ ] 页面截图为真实系统截图
- [ ] 文案与当前系统页面一致
- [ ] 没有使用明显过期的旧流程说明
- [ ] 没有引用员工当前无法访问的入口
- [ ] 没有要求员工执行超权限动作

## 二、考试准备检查

### 1. 题库是否完成第一版

- [ ] 单选题已准备
- [ ] 判断题已准备
- [ ] 场景题已准备
- [ ] 总题量已控制在 20 到 30 题

### 2. 考试规则是否确认

- [ ] 总分 100 分
- [ ] 合格线 80 分
- [ ] 第一次考试规则已设置
- [ ] 补考次数已设置
- [ ] 考试时长已设置

### 3. 考试内容是否与课件一致

- [ ] 题目不超出培训课件范围
- [ ] 重点覆盖任务、通知、查询、留痕、异常升级
- [ ] 没有纯记忆型无效题
- [ ] 没有明显歧义题

## 三、飞书学习平台准备检查

- [ ] 飞书课程已创建
- [ ] 六个课程单元已建立
- [ ] 资料上传完成
- [ ] 学习顺序已设置
- [ ] 结业考试已创建
- [ ] 学员访问权限已配置
- [ ] 试点人员已加入学习范围

## 四、ERP 侧管理准备检查

在完整培训模块开发前，ERP 侧先至少保证管理口径可落地。

- [ ] 已定义培训项目基础台账
- [ ] 已定义试点人员名单
- [ ] 已定义员工所属部门 / 岗位 / 门店口径
- [ ] 已定义结果回填模板
- [ ] 已定义完成状态口径
- [ ] 已定义未通过、逾期、补考的处理口径

建议先使用以下状态：
- `待学习`
- `学习中`
- `待考试`
- `已通过`
- `未通过`
- `已逾期`

## 五、组织分工检查

### 1. 角色分工是否明确

- [ ] 培训负责人已确定
- [ ] 课件制作负责人已确定
- [ ] 飞书搭建负责人已确定
- [ ] ERP 结果台账负责人已确定
- [ ] 主管确认负责人已确定

### 2. 每个角色职责是否明确

建议职责如下：

培训负责人：
- 统筹试点进度
- 确认培训规则
- 审核培训内容

课件制作负责人：
- 整理 PPT / 讲义
- 校验截图和页面一致性

飞书搭建负责人：
- 创建课程
- 配置考试
- 管理学员范围

ERP 台账负责人：
- 维护试点名单
- 回填学习结果
- 输出完成情况

直属主管：
- 督促学员完成
- 确认异常和补训安排

## 六、试点学员准备检查

- [ ] 试点名单已确认
- [ ] 每位试点员工有可用账号
- [ ] 每位试点员工能访问飞书学习内容
- [ ] 每位试点员工知道培训开始和结束时间
- [ ] 每位试点员工知道考试规则
- [ ] 每位试点员工知道遇到问题找谁

## 七、试点上线前最后确认

上线前建议做一次小范围预演。

- [ ] 用 1 个测试账号完整走一遍学习流程
- [ ] 用 1 个测试账号完整走一遍考试流程
- [ ] 验证考试结果能被记录
- [ ] 验证负责人能看到完成结果
- [ ] 验证补考规则可执行
- [ ] 验证异常问题有处理路径

## 八、试点成功标准

本轮试点成功，不看页面数量，主要看下面几项：

- [ ] 员工可以正常完成全部学习内容
- [ ] 员工可以正常参加考试
- [ ] 培训负责人可以区分已完成 / 未完成 / 未通过
- [ ] 主管可以拿到未完成名单
- [ ] 能识别本轮流程中的主要阻塞点

## 九、试点后复盘项

试点结束后必须复盘以下问题：

- [ ] 哪些课件内容不清楚
- [ ] 哪些题目存在歧义
- [ ] 哪些页面截图已经过期
- [ ] 员工最容易卡在哪个环节
- [ ] 哪些结果统计依然依赖人工
- [ ] 下一轮是否加入 ERP 任务 / 通知联动

## 十、当前建议

如果时间紧，优先保证下面四项，不要因为追求完整而拖延试点：

- [ ] 课件能学
- [ ] 题目能考
- [ ] 名单能管
- [ ] 结果能看

第一轮试点的目标是跑通闭环，不是一次性做到完美。
