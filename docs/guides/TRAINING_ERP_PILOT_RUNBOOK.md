# ERP 操作培训第一期试点执行与结果回填 Runbook

## 目的

本 Runbook 用于指导 `ERP 操作培训第一期` 的真实试点执行，确保在完整培训系统上线前，团队也能按统一口径完成：
- 试点发放
- 学习跟踪
- 考试结果收集
- 结果回填
- 未通过与逾期处理

## 一、试点角色分工

### 1. 培训负责人

职责：
- 确认试点名单
- 发布培训通知
- 跟踪试点进度
- 组织复盘

### 2. 飞书课程管理员

职责：
- 配置课程与考试
- 维护学员访问权限
- 导出学习与考试结果

### 3. ERP 台账负责人

职责：
- 维护试点人员台账
- 回填学习结果
- 输出完成率、未通过名单、逾期名单

### 4. 直属主管

职责：
- 督促员工按时完成
- 协助处理异常情况
- 确认是否安排补考或补训

## 二、试点执行标准流程

建议按下面 7 步执行。

### 第 1 步：确认试点名单

最低需要收集以下信息：
- 员工姓名
- 员工编号
- 所属部门
- 所属岗位
- 直属主管
- 飞书账号
- 培训开始日期
- 培训截止日期

### 第 2 步：发布试点通知

通知内容至少包括：
- 培训名称
- 培训目的
- 学习入口
- 考试要求
- 截止时间
- 遇到问题联系谁

### 第 3 步：开放学习与考试

培训负责人确认：
- 所有学员都能访问课程
- 所有学员都能参加考试
- 试点周期已经清楚告知

### 第 4 步：中途跟踪进度

建议至少做两次跟踪：
- 截止前 2 到 3 天
- 截止当天

跟踪口径建议分为：
- 未开始
- 学习中
- 已考试未通过
- 已通过

### 第 5 步：导出学习与考试结果

建议从飞书侧导出：
- 学员姓名
- 学习完成情况
- 考试成绩
- 是否通过
- 完成时间

### 第 6 步：回填 ERP 管理台账

在完整培训模块上线前，建议先用统一台账回填。

建议台账字段：
- 培训项目
- 员工姓名
- 员工编号
- 部门
- 岗位
- 直属主管
- 开始日期
- 截止日期
- 学习状态
- 考试成绩
- 是否通过
- 完成时间
- 当前状态
- 备注

### 第 7 步：处理异常与复盘

需要单独记录：
- 无法进入课程
- 无法参加考试
- 题目歧义
- 页面截图与真实系统不一致
- 培训内容看不懂
- 超期未完成

## 三、状态口径建议

为了避免试点期间统计口径混乱，建议统一使用以下状态：

### 1. 待学习

定义：
- 已纳入试点名单
- 尚未开始学习

### 2. 学习中

定义：
- 已开始学习
- 尚未完成全部学习内容或尚未参加考试

### 3. 待考试

定义：
- 学习内容完成
- 尚未参加考试

### 4. 已通过

定义：
- 完成学习
- 考试达到合格线

### 5. 未通过

定义：
- 已参加考试
- 成绩未达到合格线

### 6. 已逾期

定义：
- 超过截止时间仍未达到通过标准

## 四、结果回填频率建议

第一轮试点建议不要等结束后一次性回填，至少做 3 次：

- 第一次：培训发放当天，建立基础名单
- 第二次：培训中期，更新学习进度
- 第三次：截止后，更新最终结果

如果试点人数较少，也可以每天更新一次。

## 五、未通过处理建议

### 情况 1：学习未完成

处理建议：
- 先补完学习内容
- 再进入考试

### 情况 2：考试未通过

处理建议：
- 允许补考一次
- 补考前建议重新回看重点模块

### 情况 3：两次仍未通过

处理建议：
- 列入补训名单
- 由直属主管或培训负责人做定向辅导

## 六、逾期处理建议

对逾期学员建议按以下顺序处理：

1. 先确认是否技术问题导致
2. 确认是否时间安排问题
3. 确认是否内容理解问题
4. 视情况安排补学、补考或主管跟进

不要只记录“未完成”，要尽量记录未完成原因。

## 七、试点复盘模板

试点结束后建议至少复盘以下问题：

### 内容侧

- 哪个模块最难理解
- 哪张截图最容易过期
- 哪些说明不够清楚

### 考试侧

- 哪些题存在歧义
- 哪些题过难或过易
- 通过率是否合理

### 执行侧

- 哪一步最容易卡住
- 学员最常见问题是什么
- 管理侧最难统计的是什么

### 系统侧

- 哪些流程已经适合接入 ERP 培训管理模块
- 哪些地方还只能靠人工维护
- 下一轮是否要接任务中心和通知中心

## 八、第一轮试点的最小成功标准

当以下条件满足时，可视为第一轮试点有效：

- 大部分学员可以顺利进入课程
- 大部分学员可以顺利完成考试
- 负责人能明确区分通过 / 未通过 / 逾期
- 能输出试点完成情况名单
- 能形成下一轮优化清单

## 九、推荐下一步

如果第一轮试点跑通，下一步建议按顺序推进：

1. 把结果台账固化为 ERP 培训项目口径
2. 把试点名单、状态、结果搬到 ERP 页面
3. 再接入任务中心和通知中心
4. 然后扩展到运营或客服岗位认证培训

先跑通流程，再做系统化，是当前阶段最稳的路径。
