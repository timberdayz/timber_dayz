# 统一店铺身份口径原则

## 目的

本文用于统一西红 ERP 中不同数据域的店铺归属口径，避免：

- 同一店铺在不同模块中显示为不同身份
- `orders`、`analytics`、月结中心、业务概览之间口径不一致
- 文件级占位店铺ID被误当作真实店铺结果

## 核心原则

### 1. 标准店铺身份唯一来源

系统内唯一标准店铺身份主数据来源为：

- `core.shop_accounts`

标准店铺应以以下字段作为正式身份：

- `platform_shop_id`
- `shop_account_id`
- `store_name`

### 2. 不同数据域允许不同归属入口

不同数据域可以使用不同的原始归属入口：

- `orders` 数据域
  - 使用原始 `店铺` 标签作为入口
  - 通过 `shop_account_aliases` 归属到标准店铺
- `analytics / 流量` 数据域
  - 使用源 `shop_id` 直接归属
- 其他数据域
  - 使用该数据域最稳定、最可信的原始店铺字段归属

### 3. 统一的是最终店铺身份，不是原始入口字段

不要求所有数据域都使用相同原始字段。

要求的是：

- 所有数据域最终都必须落到同一个标准店铺身份
- 下游聚合层、展示层、月结层只消费标准店铺结果

### 4. `shop_account_aliases` 是 `orders` 店铺归属的正式机制

对于 `orders` 数据域：

- 原始店铺标签
- `shop_account_aliases`
- 标准店铺

是正式归属链路，不是临时补丁。

只要原始标签被认领并写入别名表，刷新数据链路后必须自动生效。

### 5. 文件级 `shop_id` 仅作兜底

文件级 `shop_id`（例如 `xihong`）只允许作为：

- 文件级占位值
- 入库前的兜底上下文

不允许作为最终店铺归属结果持续向下游透传。

处理规则：

- 命中标准店铺时，必须替换为标准店铺身份
- 未命中标准店铺时，应落为 `unknown`

### 6. `unknown` 是数据治理信号

`shop_id = unknown` 不代表业务结果，只代表：

- 原始店铺标签未认领
- 原始店铺标签缺失
- 当前解析规则未覆盖

`unknown` 数据应进入持续治理和监控，不应长期视为正常业务归属结果。

### 7. 新增别名后应自动生效

只要新增并认领了店铺别名，刷新以下链路后应自动归属：

- `semantic.fact_orders_atomic`
- `semantic.fact_orders_monthly_atomic`
- `mart.shop_*_kpi`
- `api.business_overview_*`
- 月结中心与相关页面

新增别名不应要求再次修改代码。

### 8. 下游模块只消费标准店铺身份

以下模块不应直接消费原始店铺标签：

- 业务概览
- 月结中心
- 店铺利润口径
- 店铺工作区
- 其他聚合与展示模块

这些模块应只消费：

- 已统一解析后的标准店铺身份

## 数据域口径示例

### `orders`

归属链路：

`原始店铺标签 -> shop_account_aliases -> shop_accounts -> 标准店铺`

### `analytics`

归属链路：

`源 shop_id -> shop_accounts -> 标准店铺`

### 聚合层 / 页面层

消费口径：

`标准店铺身份`

## 运营与治理要求

### 1. 别名认领

当原始数据出现新的店铺标签时，应：

- 进入待认领清单
- 认领到标准店铺
- 刷新数据链路验证是否生效

### 2. unknown 监控

应持续监控：

- `shop_id = unknown`
- 有原始店铺标签但未命中的记录
- 文件级占位值未被拆分的记录

### 3. 变更约束

任何新模块如果需要店铺归属：

- 必须接入标准店铺身份口径
- 不允许自行定义另一套店铺归属规则

## 一句话原则

允许不同数据域使用不同入口字段归属，但所有数据域最终必须统一落到 `core.shop_accounts` 定义的标准店铺身份。
