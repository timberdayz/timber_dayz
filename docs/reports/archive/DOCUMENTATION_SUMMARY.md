## 📋 文档管理规则

### 🎯 核心原则
1. **分层管理**: 核心文档在根目录，分类文档在子目录
2. **突出重要**: 重要文档在根目录，便于快速访问
3. **定期归档**: 过期文档自动归档到archive/
4. **标准命名**: 使用英文，kebab-case格式

### 📁 分类规则
- **核心文档**: 用户和开发者必读，保留在根目录
- **操作指南**: 用户操作相关，移至guides/
- **架构文档**: 技术设计相关，移至architecture/
- **开发文档**: 开发流程相关，移至development/
- **报告文档**: 项目进度相关，移至reports/
- **归档文档**: 过期文档，按时间分类到archive/
