# Products Semantic Template Design

**Goal**

让 `products` 数据域从“原始表头强一致”转为“核心语义一致”，避免平台把 `已付款订单 / 已确认订单 / 已确定订单` 这类同义业务字段改名时阻断同步。

**Current Problem**

- `shopee/products/monthly` 的新文件存在两种业务字段版本：
  - `已付款订单`
  - `已确认订单 / 已确定订单`
- 当前模板治理按原始表头逐字比较，导致即便语义相同，也会被判为 `update_required`。
- 文件页按文件粒度展示漂移，因此会出现待同步文件被拦截；模板页按组合最新样本文件展示，可能不显示需要更新，进一步放大理解成本。

**Design**

1. 为 `products` 引入“语义同义字段”规则，而不是继续拆更多模板。
2. 第一阶段只覆盖已经确认的 Shopee products 业务同义字段：
   - `销售额（已确认订单）` / `销售额（已确定订单）` -> `销售额（已付款订单）`
   - `订单转化率（已确认订单）` / `订单转化率（已确定订单）` -> `订单转化率（已付款订单）`
   - `已确认订单` / `已确定订单` -> `已付款订单`
   - `件数（已确认订单）` / `件数（已确定订单）` -> `件数（已付款订单）`
   - `买家数（已确认订单）` / `买家数（已确定订单）` -> `买家数（已付款订单）`
   - `转化率（已确认订单）` / `转化率（已确定订单）` -> `转化率（已付款订单）`
   - `每笔订单销售额（已确认订单）` / `每笔订单销售额（已确定订单）` -> `每笔订单销售额（已付款订单）`
   - `订单复购率（已确认订单）` / `订单复购率（已确定订单）` -> `订单复购率（已付款订单）`
   - `订单复购的平均天数（已确认订单）` / `订单复购的平均天数（已确定订单）` -> `订单复购的平均天数（已付款订单）`
3. 模板匹配阶段继续保留：
   - 原始表头 exact match
   - 语义 alias match
4. 当 exact match 失败但 semantic match 成功时：
   - 文件状态应为 `alias_only`
   - `should_auto_sync = true`
   - 不再阻断同步

**Why This Scope**

- 这是最小可交付修复，能直接解决当前 `products` 新文件被误判为需要更新模板的问题。
- 用户提到的“字段白名单 + 核心 hash 字段配置”是下一阶段治理方向，但不适合和这次问题一起改动。

**Future Direction**

下一阶段可继续演进到：
- 模板字段白名单：用户决定哪些字段需要入库
- 核心 hash 字段配置：用户决定哪些字段参与唯一性/UPSERT
- 将同义字段语义治理扩展到 `analytics / services / orders`
