# 目标管理与绩效管理前端重构设计

## 目标

重构当前 `TargetManagement.vue` 与 `PerformanceManagement.vue` 的前端信息架构，使“店铺目标 / 个人目标 / 运营目标 / 绩效结果”边界清晰，避免当前单页大表单、语义混杂、操作反人类的问题。

本次设计聚焦前端结构，不重新定义后端工资公式，但要求前端与新的“店铺绩效 / 个人绩效 / 运营目标分层”保持一致。

## 当前主要问题

### 1. 一个页面承载过多语义

当前 `TargetManagement.vue` 同时承担：

- 常规月度店铺目标
- 产品目标
- 战役目标
- 运营目标
- 店铺拆解
- 时间拆解

问题：

- 用户先要理解目标类型，再理解拆解类型，再理解运营目标规则
- 店铺目标和个人目标没有清晰边界
- 运营目标像是在填“规则引擎配置”，而不是业务目标

### 2. 运营目标表单复杂且解释弱

当前运营目标配置包含：

- 指标
- 方向
- 目标值 / 实际值
- 满分
- 罚分开关
- 罚分阈值
- 每单位罚分
- 最大罚分
- 人工评分开关
- 人工打分值

问题：

- 一次性暴露太多字段
- 用户不知道该目标属于店铺还是个人
- 用户不知道配置后会影响哪一层绩效或收入

### 3. 目标配置页与绩效结果页职责交叉

当前：

- `TargetManagement` 既像配置中心，又像部分规则中心
- `PerformanceManagement` 既展示店铺绩效，也混入个人调整项

问题：

- “配置”和“结果”没有真正分离
- 业务人员很难建立稳定心智模型

## 新的信息架构

### 一级结构

建议将当前目标与绩效相关前端入口拆成 4 类：

1. 店铺目标
2. 个人目标
3. 绩效结果
4. 规则中心

### 路由建议

#### 目标管理

- `/target-management/shop`
  - 店铺目标管理
- `/target-management/person`
  - 个人目标管理
- `/target-management/operation`
  - 运营目标管理

#### 绩效管理

- `/hr-performance-management/shop`
  - 店铺绩效结果
- `/hr-performance-management/person`
  - 个人绩效结果

#### 规则中心

- `/performance-rules`
  - 绩效规则与评分说明

说明：

- 保留原 `/target-management` 与 `/hr-performance-management` 作为兼容入口
- 兼容入口应重定向到推荐默认子页，而不是继续承载复杂逻辑

## 页面职责拆分

### 1. 店铺目标管理页

职责：

- 维护店铺月度销售目标
- 维护店铺利润目标
- 维护重点商品目标
- 维护店铺级运营目标
- 维护店铺拆解与时间拆解

页面展示：

- 顶部：月份筛选、平台筛选、店铺筛选、创建按钮
- 中部：店铺目标列表
- 右侧抽屉：店铺目标编辑器
- 底部或二级面板：拆解视图（日/店）

不再承担：

- 个人目标配置
- 个人运营目标配置

### 2. 个人目标管理页

职责：

- 维护员工月度个人目标
- 维护个人销售目标
- 维护个人运营任务目标
- 维护人工绩效奖惩

页面展示：

- 顶部：月份筛选、部门/员工筛选、创建按钮
- 中部：个人目标列表
- 右侧抽屉：个人目标编辑器

不再承担：

- 店铺目标配置
- 店铺拆解

### 3. 运营目标管理页

职责：

- 作为“运营目标模板与规则化录入中心”
- 统一管理两类运营目标：
  - 店铺运营目标
  - 个人运营目标

页面展示：

- 顶部：作用范围筛选（店铺 / 个人）
- 中部：运营目标列表
- 右侧抽屉：运营目标编辑器
- 右侧说明区：实时公式预览与影响范围说明

不再承担：

- 其它目标类型的配置

### 4. 绩效结果页

拆为两个 tab / 子页：

#### 店铺绩效结果

展示：

- 店铺绩效四个维度得分
- 总分
- 排名
- 绩效系数
- 预警 / 观察池 / 正式池

#### 个人绩效结果

展示：

- 个人销售目标结果
- 个人运营目标结果
- 考勤影响
- 人工奖惩影响
- 个人总分
- 个人绩效系数（如启用）

说明：

- 绩效结果页只做“算完以后看结果”
- 不再承担目标定义

## 运营目标前端交互重构

### 核心原则

运营目标编辑必须从“通用大表单”改成“规则化编辑器”。

### 新交互流程

#### Step 1. 选择目标作用范围

- 店铺运营目标
- 个人运营目标

系统在页面显著位置展示：

- 本目标将影响：店铺绩效 / 个人绩效
- 本目标不会直接影响：工资单 / 提成

#### Step 2. 选择指标模板

从预设模板中选择，而不是从空白字段开始。

建议模板分类：

- 时效类
- 合规类
- 库存类
- 内容类
- 人工评分项

每个模板预置：

- 指标代码
- 指标方向
- 默认满分
- 是否支持罚分
- 是否支持人工打分

#### Step 3. 只显示必要字段

根据指标模板动态展开字段：

- 通用字段
  - 目标值
  - 实际值
  - 满分
- 若启用罚分才显示：
  - 罚分阈值
  - 每单位罚分
  - 最大罚分
- 若是人工评分项才显示：
  - 人工打分值

禁止所有字段一次性全部展示。

#### Step 4. 实时公式预览

编辑器右侧或底部实时显示：

- 当前目标属于：店铺 / 个人
- 计算方式：`higher_better` / `lower_better` / `manual_score`
- 预计得分
- 作用层：
  - 店铺绩效
  - 个人绩效
- 对收入影响：
  - 影响店铺绩效系数
  - 影响个人绩效工资
  - 仅展示，不直接影响收入

### 推荐组件形态

不要继续使用长页内大表单，改成：

- 列表页
- 右侧抽屉编辑
- 高级字段折叠
- 公式预览区固定可见

## 表格与编辑器分工

### 列表表格负责什么

列表页只展示最关键摘要：

- 目标名称
- 作用范围
- 指标类型
- 当前目标值
- 当前实际值
- 满分
- 当前得分
- 状态

### 编辑抽屉负责什么

编辑抽屉只负责：

- 录入
- 预览
- 校验

不在列表页直接铺满可编辑字段。

## 前端字段矩阵建议

### 店铺目标页

保留字段：

- 目标名称
- 月份
- 平台 / 店铺
- 销售目标
- 利润目标
- 重点商品目标
- 店铺运营目标摘要

### 个人目标页

- 个人运营目标摘要
- 考勤影响摘要

保留字段：

- 指标模板
- 指标方向
- 目标值
- 实际值
- 满分

## 页面之间的职责边界

### `TargetManagement`

重构后只保留“目标定义与维护”职责，不再承担复杂绩效解释。

### `PerformanceManagement`

重构后只保留“绩效结果展示与重算”职责，不再成为目标录入入口。

### `MyIncome`

只展示工资单结果与说明，不展示目标配置。

## 推荐实现顺序

### Phase 1

- 把当前 `TargetManagement.vue` 拆成：
  - 店铺目标页
  - 个人目标页
  - 运营目标页
- 保留原路由兼容跳转

### Phase 2

- 将运营目标表单改造成模板化抽屉式编辑器
- 增加实时公式预览

### Phase 3

- 拆 `PerformanceManagement.vue` 为店铺绩效 / 个人绩效双视图
- 将“个人绩效调整项”迁入个人绩效视图

### Phase 4

- 清理旧的大表单逻辑
- 下掉旧的多语义共享字段与分支

## 结论

最终前端必须从：

```text
一个大页面 + 多种目标混杂 + 长表单硬填
```

重构成：

```text
按对象分流的页面 + 列表/抽屉结构 + 模板化规则编辑器 + 实时公式预览
```

只有这样，运营目标在拆分为“店铺运营目标”和“个人运营目标”之后，前端体验才会真正合理，而不是把复杂度继续压给业务用户。
