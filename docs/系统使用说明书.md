# 西虹ERP系统使用说明书

**版本**: v1.0  
**更新日期**: 2026-02-02  
**适用对象**: 业务、运营、数据、人力资源等使用系统的团队成员

---

## 一、总览与数据流

### 1.1 系统是做什么的

西虹ERP是**跨境电商ERP系统**，支撑「**配置目标 → 采集数据 → 自动计算指标 → 看板/报表展示**」的完整链路，用于：

- **经营**：看整体业绩是否达标、和上期比如何、各店铺/平台谁完成得好。
- **运营**：看流量、转化、库存、清仓、店铺日常表现，指导执行动作。
- **人力资源**：看绩效公示、个人收入、人员店铺归属与提成。

### 1.2 数据从哪来

- **A类（用户配置）**：在系统里直接配置，如销售战役、月度目标、目标分解、绩效权重。  
  → 入口：**销售与分析 → 目标管理 / 销售战役管理**；**人力资源 → 人员店铺归属和提成比** 等。
- **B类（业务数据）**：从电商平台导出 Excel，经**字段映射/数据同步**入库（订单、产品、流量、库存等）。  
  → 入口：**数据采集与管理 → 数据同步（文件列表 / 同步任务 / 模板管理）**。
- **C类（计算数据）**：系统根据 A 类 + B 类自动计算达成率、排名、健康度、经营指标等。  
  → 主要在**工作台 → 业务概览**及 Metabase 报表中展示。

整体流程可概括为：

```
用户配置(A类) → 数据采集(B类) → 系统计算(C类) → 前端展示
```

更详细的说明见：`docs/CORE_DATA_FLOW.md`。

### 1.3 指标主要在哪看

- **工作台 → 业务概览**：核心 KPI、数据对比、店铺赛马、经营指标、流量排名、库存积压、清仓排名等。
- **销售与分析**：销售看板、目标管理、销售战役管理、订单管理。
- **产品与库存**：库存管理、库存看板、库存健康仪表盘、产品质量仪表盘。
- **人力资源**：绩效公示、我的收入、人员店铺归属和提成比、绩效管理（管理员）。
- **财务管理**：财务总览、应收应付、费用管理、财务报表等。
- **Metabase**：深度分析、自定义报表（若已部署，通常通过 Nginx 子路径如 `/metabase/` 访问）。

## 二、指标与经营/运营需求对照

下表说明：**具体经营/运营需求** 对应 **系统里的哪些指标**，以及 **在哪个入口查看**。

| 需求类型 | 具体需求 | 系统指标 | 数据来源/看板 | 入口（菜单位置） |
|----------|----------|----------|----------------|------------------|
| **经营** | 看整体业绩是否达标 | 月目标、达成率、时间 GAP、今日销售、经营结果 | 经营指标 | 工作台 → 业务概览（经营指标区块） |
| **经营** | 看 GMV/订单/利润是否健康 | GMV、订单数、客单价、连带率、人效 | 核心 KPI | 工作台 → 业务概览（核心 KPI 卡片） |
| **经营** | 看和上期/平均比是好是差 | 本期/上期/平均、环比、目标达成率 | 数据对比 | 工作台 → 业务概览（数据对比 + 目标达成率） |
| **经营** | 看各店铺/平台谁完成得好 | 按店铺/平台的目标、完成额、完成率、排名 | 店铺赛马 | 工作台 → 业务概览（店铺赛马） |
| **经营** | 目标设定与跟踪 | 销售目标、目标分解、达成情况 | 目标管理 + 业务概览 | 销售与分析 → 目标管理；业务概览（数据对比/经营指标） |
| **经营** | 销售战役效果 | 战役目标、参与店铺、进度 | 销售战役 | 销售与分析 → 销售战役管理 |
| **运营** | 流量是否足够、谁流量高 | 访客数、浏览量、排名 | 流量排名 | 工作台 → 业务概览（流量排名） |
| **运营** | 转化好不好 | 转化率 | 核心 KPI | 工作台 → 业务概览（核心 KPI） |
| **运营** | 库存是否积压 | 积压商品、积压天数 | 库存积压 | 工作台 → 业务概览（库存积压） |
| **运营** | 清仓/滞销品优先级 | 清仓排名（滞销分析） | 清仓排名 | 工作台 → 业务概览（清仓排名）或 Metabase |
| **运营** | 店铺/账号日常表现 | 按店铺的销售、排名、健康度 | 店铺赛马 + 店铺分析 | 工作台 → 业务概览；店铺运营 → 店铺管理/店铺分析 |
| **人力资源** | 绩效是否公平、可解释 | 绩效得分、排名、系数 | 绩效公示 | 人力资源 → 绩效公示 |
| **人力资源** | 个人收入明细 | 底薪、绩效、提成、实发（规划中/部分上线） | 我的收入 | 人力资源 → 我的收入 |
| **人力资源** | 谁负责哪家店、提成比例 | 店铺归属、提成比 | 人员店铺归属 | 人力资源 → 人员店铺归属和提成比 |

**说明**：

- **经营**：侧重目标、达成、时间进度、费用/毛利、结果判断（要不要调目标、调资源）。
- **运营**：侧重流量、转化、库存、清仓、店铺日常执行（今天推哪家店、清哪些货）。

## 三、按角色的使用路径

### 3.1 经营负责人 / 老板

1. 登录系统。
2. 进入 **工作台 → 业务概览**。
3. 查看：
   - **核心 KPI**：GMV、订单数、客流量、转化率、客单价、连带率、人效（及环比）。
   - **数据对比**：选择日/周/月 + 日期，看本期、上期、平均、环比及目标达成率。
   - **经营指标**：月目标、达成率、时间 GAP、今日销售、经营结果。
   - **店铺赛马**：各店铺/平台的目标、完成额、完成率、排名。
4. 需要设定或调整目标时：**销售与分析 → 目标管理**。
5. 需要看战役效果时：**销售与分析 → 销售战役管理**。

### 3.2 运营 / 店长

1. **工作台 → 业务概览**：
   - 流量排名、库存积压、清仓排名。
   - 核心 KPI（转化率等）、店铺赛马。
2. **销售与分析**：销售看板、目标管理、订单管理。
3. **产品与库存**：库存管理、库存看板、库存健康仪表盘、产品质量仪表盘。
4. **店铺运营**：店铺管理、店铺分析、账号管理、账号对齐。

### 3.3 数据 / 采集同事

1. **数据采集与管理**：
   - **数据同步 → 文件列表 / 同步任务 / 同步历史 / 模板管理**：Excel 入库、字段映射、同步进度。
   - **数据隔离区**：异常数据排查。
   - **数据一致性验证**：跨平台一致性、计算数据验证、异常检测。
2. 采集配置、采集任务、采集历史、组件录制/版本管理（若启用）。
3. 数据上来后，业务概览等看板的指标才会更新，因此需保证同步任务与模板配置正确。

### 3.4 人力资源 / 绩效

1. **人力资源**：
   - **绩效公示**：全员可见的绩效得分、排名、系数。
   - **我的收入**：员工自助查看收入明细（底薪、绩效、提成等，视功能上线情况）。
   - **人员店铺归属和提成比**：谁负责哪家店、提成比例。
   - **绩效管理**：管理员进行绩效计算与配置（若仅管理员可见）。

## 四、关键页面与操作要点

### 4.1 业务概览页

- **入口**：左侧菜单 **工作台 → 业务概览**。
- **核心 KPI 卡片**：GMV、订单数、客流量、转化率、客单价、连带率、人效；支持按月、平台筛选，部分指标带环比。
- **数据对比**：选择粒度（日度/周度/月度）、日期、平台；查看本期、上期、平均、环比及目标达成率；周度时请选择该周周一日期。
- **店铺赛马**：按店铺或平台查看目标、完成额、完成率、排名。
- **经营指标**：月目标、达成率、时间 GAP、预估费用、今日销售、今日销售单数、经营结果；选择月份、平台。
- **流量排名**：访客数、浏览量排名；可选粒度、维度、日期、平台、店铺。
- **库存积压**：积压商品、积压天数；可设积压天数阈值、平台、店铺。
- **清仓排名**：滞销/清仓商品分析；可选日期范围、平台、店铺、返回数量。
- **连带率趋势**：可选平台、粒度、日期范围，查看连带率趋势图。

### 4.2 目标管理

- **入口**：**销售与分析 → 目标管理**。
- **作用**：维护销售目标、目标分解（按店铺/时间）；业务概览的「数据对比」「经营指标」「店铺赛马」会依赖这些目标数据。
- **操作**：创建/编辑目标、设置周期与分解，保存后可在业务概览中看到达成率等指标更新（依赖数据同步与计算刷新）。

### 4.3 数据同步（字段映射与入库）

- **入口**：**数据采集与管理 → 数据同步 → 文件列表 / 同步任务 / 模板管理**。
- **作用**：将平台导出的 Excel 通过字段映射识别并入库，形成订单、产品、流量、库存等 B 类数据；只有这里数据正确入库，业务概览等 C 类指标才有数。
- **要点**：选择平台与数据域、配置或选择模板、执行同步；可查看任务日志、数据流转、原始数据对比（见 `docs/V4_11_5_USER_GUIDE.md`）。

### 4.4 绩效公示与我的收入

- **绩效公示**：**人力资源 → 绩效公示**，查看绩效得分、排名、系数等（全员可见）。
- **我的收入**：**人力资源 → 我的收入**，员工查看本人收入明细（功能视上线情况而定，若暂无数据可参考「常见问题」）。

## 五、指标口径与计算说明（附录）

| 指标 | 口径说明 |
|------|----------|
| **GMV** | 实付金额汇总（订单明细中的 paid_amount）。 |
| **订单数** | 去重订单数（order_id）。 |
| **客流量** | 访客数（来自流量/分析数据）。 |
| **转化率** | 订单数 / 访客数（或按 Metabase Question 中定义）。 |
| **客单价** | GMV / 订单数。 |
| **连带率** | 订单商品件数 / 订单数。 |
| **人效** | GMV（或指定指标）/ 在职人数（a_class.employees 中 status=active）。 |
| **目标达成率** | 实际完成值 / 目标值；经营指标中的「时间 GAP」等以系统当前 Question 定义为准。 |
| **日/周/月** | 日度=单日；周度=周一～周日（选周一时段）；月度=当月 1 日～月末。 |

数据来源与 SQL 定义见：`sql/metabase_questions/` 下对应 Question、以及 `config/metabase_config.yaml`。

## 六、常见问题

**Q1：业务概览里 KPI 或对比是空的？**  
先确认：① 是否已在「数据同步」中完成对应平台/时间段的 Excel 入库；② 是否在目标管理中配置了目标（影响达成率、经营指标）。数据同步与目标配置正确后，指标会按当前计算逻辑展示。

**Q2：周度对比选哪一天？**  
选择该周的**周一**日期，系统按周一～周日为一周统计。

**Q3：绩效公示显示「查询绩效评分列表失败」或为空？**  
可能原因：绩效尚未计算、或计算依赖的目标/店铺数据未就绪。需管理员在「绩效管理」中执行计算（若已开放）；同时可参考 `openspec/changes/add-performance-and-personal-income/` 相关说明。

**Q4：我的收入页面没有数据？**  
若功能已上线但无数据，可能尚未配置工资单或绩效/提成数据源；具体以当前版本功能说明为准。

**Q5：清仓排名、库存积压在哪里？**  
在**工作台 → 业务概览**页内；若部署了 Metabase，也可在 Metabase 的「C类数据报表/业务概览」或清仓相关 Question 中查看。

**Q6：Metabase 怎么访问？**  
若已配置 Nginx 反向代理，通常通过浏览器访问：`https://你的域名/metabase/`（或运维提供的地址）。
