## 修改的需求

### 需求：看板数据加载可靠性
所有看板页面必须可靠地加载数据，不频繁出现"加载失败"错误。

#### 场景：库存管理页面成功加载
- **当** 用户导航到库存管理页面时
- **则** 页面必须无错误地加载库存列表
- **且** 如果数据加载失败，显示包含恢复建议的用户友好错误消息

#### 场景：销售看板成功加载
- **当** 用户导航到销售看板页面时
- **则** 页面必须无错误地加载销售统计和产品排名
- **且** 如果数据加载失败，显示包含恢复建议的用户友好错误消息

#### 场景：库存健康仪表盘成功加载
- **当** 用户导航到库存健康仪表盘页面时
- **则** 页面必须无错误地加载健康指标
- **且** 如果数据加载失败，显示包含恢复建议的用户友好错误消息

## 新增的需求

### 需求：错误恢复建议
所有数据加载错误必须包含针对用户的恢复建议。

#### 场景：数据库字段错误恢复
- **当** 发生数据库字段错误时
- **则** 错误消息必须包含：
  - 明确指示字段缺失
  - 建议刷新页面或联系支持
  - 面向开发者的技术细节（在控制台日志中）

#### 场景：API方法缺失错误恢复
- **当** 前端API方法缺失时
- **则** 错误消息必须包含：
  - 明确指示API方法不可用
  - 建议刷新页面或联系支持
  - 面向开发者的技术细节（在控制台日志中）
