# Change: 业务概览页面全局日期组件与模块同步

## Why

业务概览页面各模块（数据对比、店铺赛马、流量排名、经营指标等）均使用独立的日期选择组件，用户需要在多个位置重复选择粒度和日期，且切换时各模块数据不同步，体验分散。目标管理中「自动+手动」的设计（修改某行后锁定、未修改行自动均分）已被验证有效，可复用该模式实现：全局日期驱动多数模块，同时允许单模块手动覆盖。

**根本原因**：各模块日期组件独立实现，缺少统一的「主控」日期与同步机制。

## What Changes

### 核心变更

1. **新增页面级全局日期组件**
   - 在业务概览页面顶部（副标题下方）增加「全局日期」区域
   - 组件包含：粒度选择（日/周/月）+ 日期选择器
   - 默认：打开页面时，粒度=月，日期=当前月份（YYYY-MM）

2. **模块同步规则（自动跟随）**
   - **数据对比、店铺赛马、流量排名**：默认跟随全局，粒度+日期与全局一致
   - **经营指标**：默认跟随全局，按全局粒度映射为单日参数（日→所选日期；周→该周周一；月→该月1日）
   - **核心 KPI**：可选跟随，当全局为「月」时同步为同一月份
   - **清仓排名**：可选跟随，月/周可分别与全局月/周对齐

3. **手动覆盖机制（类似目标管理锁定）**
   - 每个模块的日期旁提供「跟随全局」开关（如链接图标）
   - 默认：跟随全局（useGlobalDate[module]=true）
   - 用户手动修改某模块日期/粒度 → 该模块设为不跟随（useGlobalDate[module]=false），不再受全局更新影响
   - 点击「跟随全局」→ 恢复跟随，并同步全局日期

4. **时间约定（与现有实现一致）**
   - 周 = 周一～周日（PostgreSQL `date_trunc('week')`）
   - 月 = 1日～月末
   - 经营指标 API 仅接收单日，同步时需映射：周→周一，月→1日

### 技术细节

- **状态**：`globalDate`（粒度+日期）、`useGlobalDate: Record<string, boolean>`
- **数据流**：全局变更 → 遍历 `useGlobalDate[module]=true` 的模块 → 更新模块 date/granularity → 调用 `loadXxxData()`
- **模块变更**：用户修改模块日期 → 设置 `useGlobalDate[module]=false` → 调用 `loadXxxData()`
- **日期格式**：沿用各模块现有 value-format，避免 toISOString UTC 偏差
