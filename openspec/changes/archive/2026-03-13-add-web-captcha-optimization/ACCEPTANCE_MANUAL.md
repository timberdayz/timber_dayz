# 人工验收清单（网页登录验证码优化）

以下需在真实环境由人工执行并打勾确认。

## 6.1 妙手图形验证码（需有验证码环境）

- [ ] 使用会触发**图形验证码**的妙手账号（或等平台随机出现验证码）
- [ ] 在前端「采集任务」页创建妙手采集任务并开始采集
- [ ] 登录阶段出现验证码时，任务状态变为 **暂停**，且任务行出现「继续」按钮
- [ ] 点击「继续」后弹出验证码弹窗：**能展示验证码截图**（来自 `GET /tasks/{id}/screenshot`）
- [ ] 输入正确验证码并提交后，任务**自动继续**并完成登录（无需刷新页面、无需重新发起任务）
- [ ] 若等待超过约 5 分钟未提交，任务应变为**失败**；此时再点「继续」并提交验证码，应看到** 4xx 提示**（如「验证已超时或任务已结束，请重新发起采集」）

## 6.2 TikTok/Shopee OTP（需有 OTP 环境）

- [ ] 使用会触发**短信/邮箱二次验证**的 TikTok 或 Shopee 账号
- [ ] 创建对应平台采集任务并开始采集
- [ ] 出现 OTP 时任务状态变为 **暂停**
- [ ] 点击「继续」后弹窗文案为「需要短信/邮箱验证码」，输入 OTP 并提交
- [ ] 任务在同一会话内继续并完成登录；与「提交验证码」API 行为一致（请求体为 `{ "otp": "..." }`）

## 可选快速检查（无验证码时）

- [ ] 创建妙手采集任务且登录**不出现**验证码时，任务能正常完成，无异常
- [ ] 任务列表/详情中能看到 `verification_type`、`verification_screenshot` 字段（暂停时才有值）

验收完成后，可在本 change 的 `tasks.md` 中将 6.1、6.2 勾选为完成。
