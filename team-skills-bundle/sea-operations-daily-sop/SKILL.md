# Sea Operations Daily SOP

## Status

This is the mandatory daily execution SOP for SEA e-commerce operations.  
It translates business standards into daily work order and time allocation.

## Core Rule

不是先问“今天做什么”，而是先问：

1. 今天有没有异常
2. 今天最重要的增长瓶颈在哪里
3. 今天应该把时间放在哪个优先级

先处理高优先级异常，再做增长动作，再做沉淀动作。

## Mandatory Gates

在安排当天工作前，必须完成：

1. 检查是否存在优先级 1 异常
2. 明确今天的主瓶颈
3. 确认今天的核心目标
4. 按角色选择对应时间分配模板
5. 如需具体钟点安排，读取 `references/time-slots.md`
6. 输出时间块和动作清单

如果没有数据支撑，不允许凭感觉安排当日重点。

## How To Invoke

当用户要求“今天运营怎么安排”“一天时间怎么分配”“运营专员每天先干什么”“负责人怎么安排团队节奏”时，必须使用本 skill。

如果是异常驱动，先引用 `sea-business-review` 或 `sea-funnel-action-playbook` 定位问题。  
如果是选品日程，引用 `sea-product-selection`。  
如果是指标优先级争议，引用 `sea-metrics-language`。

如果用户明确需要“几点做什么”，必须读取 `references/time-slots.md`。

## Standard User Request

```text
请按 team-skills-bundle/sea-operations-daily-sop/SKILL.md 为今天的运营工作制定 SOP。

要求：
1. 先判断今天是否存在优先级 1 异常
2. 再确定今天的主瓶颈
3. 按角色分配时间块
4. 每个时间块都给出明确动作
5. 输出复查节点和收尾复盘要求
```

## Output Rule

输出必须包含：

- 角色
- 当天目标
- 今日优先级
- 时间分配
- 每个时间块动作
- 复查节点
- 收尾复盘
