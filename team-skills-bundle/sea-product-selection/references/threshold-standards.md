# Threshold Standards

本文件定义选品 V0 阈值。  
V0 是团队统一起点，不是最终真理。后续必须用公司历史成功品、失败品、爆品、滞销品持续校准。

## Status

所有阈值必须标记校准状态：

- `v0`: 通用初始标准
- `calibrated`: 已用公司历史数据校准
- `needs-calibration`: 缺少足够历史数据

当前默认状态：`v0`

## 1. Demand Strength

优先使用目标国家、目标平台、目标品类内的相对分位数。

| Level | Standard | Decision |
|---|---|---|
| excellent | 搜索量位于目标类目前 25%，且近 30 / 60 天稳定或上升 | 高优先级 |
| good | 搜索量位于前 25%-50%，且有销量或内容热度支撑 | 可进入验证 |
| watch | 搜索量位于 50%-75%，但有明确痛点或高利润空间 | hold |
| weak | 搜索量位于后 25%，且无高客单、高利润、强痛点支撑 | reject |

没有分位数数据时，使用定性判断：

- 搜索低 + 无销量证据：`reject`
- 搜索中 + 有竞品销量：`hold`
- 搜索高 + 竞品弱 + 差评痛点明显：高优先级
- 搜索高 + 竞品极多：不能直接判好，继续看竞争和利润

## 2. Search Sales Ratio

搜索销售比用于识别需求与成交之间的机会或风险。

```text
搜索销售比 = 30天搜索量 / 头部竞品30天销量
```

| Ratio | Interpretation | Decision |
|---|---|---|
| `<3` | 供给可能充分，竞争偏饱和 | 继续看差异化 |
| `3-10` | 相对健康 | 可继续评估 |
| `10-30` | 可能有需求缺口 | 重点验证 |
| `>30` | 可能搜索多但购买弱 | 必须检查转化和痛点 |

禁止把 `>30` 直接判定为好机会。

## 3. Supply Competition

供给强度不能只看销量，必须综合：

- 有效竞品数量
- 头部评价壁垒
- 价格带压缩程度
- 内容竞争密度
- 同款密度

| Pattern | Decision |
|---|---|
| 搜索高 + 有效竞品少 | 高优先级 |
| 搜索高 + 有效竞品多 + 价格战严重 | 红海 |
| 搜索中 + 竞品弱 + 差评痛点明显 | 精品机会 |
| 搜索低 + 竞品少 | 可能是假机会 |

## 4. Profitability

精品选品使用贡献利润，不使用表面毛利。

```text
预计贡献利润 = 售价 - 货本 - 头程 - 尾程 - 平台费 - 优惠 - 推广/佣金 - 退款损失
```

| Contribution Margin | Decision |
|---|---|
| `>35%` | 很好，适合精品测试和内容放大 |
| `25%-35%` | 可做，适合进入测试 |
| `15%-25%` | 谨慎，只适合供应链强或复购强的品 |
| `<15%` | 不建议做精品 |
| 负利润或依赖补贴成交 | reject |

TikTok 内容型产品必须额外预留达人佣金、样品成本和内容试错成本。

## 5. Conversion Rate

转化率必须区分平台与场景。

### Shopee Shelf Commerce

| Conversion Rate | Decision |
|---|---|
| `<1%` | 弱，页面 / 价格 / 需求可能有问题 |
| `1%-2%` | 可测试 |
| `2%-5%` | 正常可做 |
| `5%-10%` | 强 |
| `>10%` | 非常强 |

### TikTok Content Commerce

| Click-to-Order Conversion | Decision |
|---|---|
| `<0.8%` | 弱 |
| `0.8%-2%` | 可继续测内容 |
| `2%-3%` | 良好 |
| `>3%` | 强 |

如果数据来自 Ads Manager、Seller Center 或 ERP，必须说明来源和归因口径。

## 6. Content Potential

TikTok 主推产品必须通过内容表现力门槛。

| Signal | Decision |
|---|---|
| 3 秒讲不清卖点 | 不适合 TikTok 放大 |
| 不能演示使用场景 | 内容表现弱 |
| 没有 before/after 或痛点解决 | 内容传播弱 |
| 达人不愿意拍 | 放大难 |
| 直播无法讲清 | 直播成交弱 |

分层：

| Level | Standard |
|---|---|
| excellent | 3 秒能讲清，能演示，能前后对比 |
| good | 能拍内容，但需要脚本优化 |
| watch | 内容弱，但 Shopee 搜索可能承接 |
| weak | 内容和搜索都弱 |

## 7. Go / Hold / Reject

| Decision | Standard |
|---|---|
| `go` | 总分 >= 80，且无一票否决，需求 / 竞争 / 利润 / 差异化至少 4 项成立 |
| `hold` | 总分 65-79，或某一关键证据不足，需要补数据 |
| `reject` | 总分 < 65，或任一一票否决项成立 |

## 8. Platform-Specific Gate

### Shopee Primary

必须满足：

- 搜索需求成立
- 页面转化有优化空间
- 评价和价格带有切入口
- 利润可以覆盖平台活动与优惠

### TikTok Primary

必须满足：

- 内容表现力成立
- 短视频 / 直播 / 达人至少一个路径成立
- 利润可以覆盖佣金、样品和内容试错

## 9. Excellent Candidate Pattern

一个非常好的精品候选品通常同时具备：

- 搜索需求稳定
- 竞品有销量但未绝对垄断
- 差评中有明确可改造痛点
- 价格带没有被打穿
- 贡献利润率足够支撑内容和投放
- 可通过包装、规格、套装或人群定位差异化
- TikTok 能拍，Shopee 能接搜索
- 供应链可控
- 售后和合规风险低
